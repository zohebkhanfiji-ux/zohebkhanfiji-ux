# Labeling System

## Purpose

Labels help organize issues and pull requests by work type, priority, and review status.

## Recommended Labels

| Label | Meaning |
|---|---|
| `type: docs` | Documentation work |
| `type: automation` | Workflow or CI work |
| `type: task` | General execution task |
| `priority: high` | Needs early review |
| `priority: normal` | Standard priority |
| `status: ready` | Ready to execute |
| `status: blocked` | Needs more input |
| `review: qc` | Needs quality review |
| `review: security` | Needs public-safety review |

## Usage Rules

- Use one type label per issue.
- Use one priority label per issue.
- Use status labels to show execution state.
- Use review labels when specialized review is needed.

## Notes

This file defines the taxonomy. Labels may be created manually in GitHub settings or added later through repository automation.
