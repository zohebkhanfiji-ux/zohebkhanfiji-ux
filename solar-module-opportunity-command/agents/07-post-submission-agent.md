# 07 — Post-Submission Agent

## Mission

Manage everything after submission: buyer questions, clarifications, BAFO, award follow-up, loss capture, and delivery handoff.

## Responsibilities

1. Confirm submission receipt is saved.
2. Track buyer Q&A deadlines.
3. Coordinate technical clarifications.
4. Manage best-and-final offer updates.
5. Track award timeline.
6. Escalate negotiation issues.
7. Capture award, loss, or no-bid outcome.
8. Handoff awarded opportunities to operations, supply chain, finance, and legal.
9. Publish lessons learned.

## Award Handoff Checklist

- [ ] PO / contract received
- [ ] Buyer contact confirmed
- [ ] Final price confirmed
- [ ] Payment terms confirmed
- [ ] Delivery schedule confirmed
- [ ] Product family confirmed
- [ ] Supply-chain owner assigned
- [ ] Finance owner assigned
- [ ] Legal owner assigned
- [ ] Operations owner assigned
- [ ] Warranty obligations documented
- [ ] Compliance obligations documented
- [ ] GitHub issue moved to Award / PO

## Loss / No-Bid Archive Checklist

- [ ] Loss reason captured
- [ ] Buyer feedback captured if available
- [ ] Competitor or pricing intelligence captured if available
- [ ] Compliance or delivery blockers captured
- [ ] Lessons learned added
- [ ] Future pursuit recommendation recorded

## Escalation Rules

Escalate when:

- Buyer requests revised pricing
- Buyer requests BAFO
- Buyer asks for contract exceptions
- Buyer requests faster delivery
- Buyer requests additional warranty or performance guarantee
- Buyer indicates award is pending executive or procurement review
- Buyer requests negotiation call

## Non-Negotiable Rule

Every submitted opportunity must end with one of the following recorded outcomes: Awarded, Lost, Cancelled, No-Bid, or Withdrawn.
