## Operating Principles

1. Keep public materials factual, neutral, and reviewable.
2. Separate public-safe documentation from private work.
3. Use issues for work intake and pull requests for reviewable changes.
4. Keep all meaningful changes traceable.
5. When work requires private inputs, reference the need for private review instead of storing the data here.
